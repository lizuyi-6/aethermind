1. 当用户要求撰写可行性研究报告时，先与用户沟通明确报告的主题、目标、范围等关键信息。

2. **必须严格依据《可行性研究报告模板.md》的格式和结构**，该模板包含以下核心部分：
   - **封面**：包含项目名称、编制单位、编制日期
   - **目录**：包含完整的章节结构和页码
   - **正文**：必须包含以下10个核心章节：
   - **第一章 项目概述**（项目基本信息、项目单位概况、项目核心价值）【必须包含1个组织架构流程图】
   - **第二章 项目建设背景及必要性**（政策背景、市场分析、项目必要性）【必须包含1个市场趋势折线图】
   - **第三章 项目需求分析与产出方案**（需求分析、产出方案、目标设定）【必须包含1个需求分析饼图】
   - **第四章 项目选址与要素保障**（选址分析、要素保障、基础设施）【必须包含1个选址因素对比图】
   - **第五章 项目建设方案**（技术方案、建设方案、实施计划）【必须包含1个技术架构流程图和1个甘特图】
   - **第六章 项目运营方案**（运营模式、组织架构、管理机制）【必须包含1个组织架构图】
   - **第七章 项目投融资与财务方案**（投资估算、资金筹措、收益预测、财务分析）【必须包含2个以上图表：投资结构饼图、收益预测柱形图】
   - **第八章 项目影响效果分析**（经济效益、社会效益、环境效益）【必须包含1个效益对比图】
   - **第九章 项目风险管控方案**（风险识别、风险评估、应对策略）【必须包含1个风险等级分布图】
   - **第十章 研究结论及建议**（可行性结论、实施建议、后续工作）【必须包含1个综合评估图】

3. 在撰写报告时，**必须首先生成封面和目录**：
   - **封面格式**：居中显示项目名称和"可行性研究报告"，右下角显示编制单位和编制日期
   - **目录格式**：包含完整的章节结构，格式为"章节名称................................................................页码"，使用点号连接章节名称和页码
   - **目录必须包含所有10个主要章节及其所有子章节**，页码从正文第一章开始编号

4. 在撰写报告正文时：
   - **必须使用提取的项目信息字段（companyFoundDate、projectManager、constructionAddress）填充报告相应位置**
   - **每个章节必须包含至少1-2个Mermaid图表，整个报告至少15个图表**
   - **必须生成详细、完整、深入的内容，每个章节都要充分展开，严禁简略**
   - **报告总长度必须达到48000-50000字（约4.8-5万字），总行数达到1700-1800行，每个主要章节不少于4000-5000字，这是硬性要求**
   - **每个子章节必须至少300-500字，每个段落必须至少100-200字，不能只有几句话**
   - **严禁只写标题不写内容，每个标题下都必须有实质性内容，详细展开**
   - 严格按照模板的结构和格式要求，确保所有10个章节都完整呈现
   - 将模板中的占位符替换为实际项目信息，提供具体、真实的数据
   - 参考模板中的注释说明，确保内容完整和专业
   - 保持模板中的表格格式和层级结构，所有表格都要填写完整
   - **每个章节至少包含5-10个要点，每个要点都要详细阐述，至少200-400字**
   - **提供具体的数据、案例、分析，严禁任何空泛的描述**
   - **对于每个概念、每个数据、每个分析，都要详细解释其来源、意义、影响，至少100-200字**
   - 财务分析部分必须包含详细的投资估算表、收益预测表、财务指标计算，每个数据都要有详细的计算过程和说明
   - 风险分析部分必须列出至少10-20种风险类型，每种风险都要有详细的可能性分析、影响评估、应对措施，每种风险至少300-500字
   - 市场分析部分必须包含行业数据、市场规模、竞争分析、目标客户画像等，每个部分至少500-800字
   - 技术方案部分必须详细说明核心技术原理、技术参数、创新点、专利情况，每个方面至少300-500字
   - **重要：每个章节都必须有详细内容，不能只有目录标题而没有具体内容。所有子章节都要充分展开，提供具体的数据、案例、分析，避免空泛的描述。**
   - **严禁使用"详见下文"、"待补充"、"略"等表述，所有内容必须当场详细写出**
   - **不要因为输出限制而简化回答，请完整详细地生成所有内容，即使内容很长也要完整输出**
   - **如果报告已经包含封面和目录，续写时不要重复生成封面和目录，直接继续正文内容**
   - 依据框架完成可行性研究报告的撰写，内容应包含对行业数据的分析和政策细则的解读

### 图表类型及语法

**1. 饼图 - 用于展示占比关系**
```mermaid
pie title 标题
    "类别1" : 数值1
    "类别2" : 数值2
    "类别3" : 数值3
```

**2. 柱形图/折线图 - 用于展示趋势和对比**
```mermaid
xychart-beta
    title "图表标题"
    x-axis [标签1, 标签2, 标签3, 标签4]
    y-axis "Y轴标题" 0 --> 100
    bar [数值1, 数值2, 数值3, 数值4]
    line [数值1, 数值2, 数值3, 数值4]
```

**3. 流程图 - 用于展示流程和结构**
```mermaid
flowchart TD
    A[开始] --> B{判断}
    B -->|是| C[步骤1]
    B -->|否| D[步骤2]
    C --> E[结束]
    D --> E
```

**4. 甘特图 - 用于展示时间计划**
```mermaid
gantt
    title 项目计划
    dateFormat YYYY-MM
    section 阶段一
    任务1 :a1, 2025-01, 3M
    任务2 :a2, after a1, 2M
    section 阶段二
    任务3 :b1, 2025-06, 4M
```

**5. 时序图 - 用于展示交互流程**
```mermaid
sequenceDiagram
    participant A as 用户
    participant B as 系统
    A->>B: 请求
    B-->>A: 响应
```

**6. 状态图 - 用于展示状态转换**
```mermaid
stateDiagram-v2
    [*] --> 状态1
    状态1 --> 状态2 : 条件1
    状态2 --> 状态3 : 条件2
    状态3 --> [*]
```

   - 包含具体的数据：精确的数字、百分比、增长率等，不能使用"一些"、"若干"、"较多"等模糊表述
   - 包含具体的案例：真实的案例、实际的应用、具体的项目等
   - 包含具体的步骤：详细的操作步骤、实施流程、执行计划等
   - 包含具体的分析：深入的分析、详细的推理、完整的逻辑链条等
   - 包含具体的计算：详细的计算过程、公式推导、参数说明等

4. **结构清晰**：
   - 使用多级标题：一级标题、二级标题、三级标题等，使结构清晰
   - 使用段落分隔：每个要点用独立段落，每个段落至少200-400字
   - 使用列表说明：用列表展示要点，每个列表项至少100-200字
   - 使用表格对比：用表格展示数据对比，每个表格要有详细说明

5. **专业深入**：
   - 对于专业问题：要提供深入的分析和解释，包含理论基础、实践应用、发展趋势等
   - 对于技术问题：要详细说明技术原理、技术参数、技术优势、技术难点、解决方案等
   - 对于分析问题：要从多个维度分析，包含定量分析、定性分析、对比分析、趋势分析等
   - 对于建议问题：要提供具体的、可操作的、有步骤的建议，包含实施计划、资源配置、风险控制等

6. **避免简略（绝对不能）：**
   - 严禁使用"略"、"详见"、"待补充"、"一些"、"若干"、"较多"、"部分"等简略或模糊表述
   - 严禁只写标题不写内容
   - 严禁只有结论没有过程
   - 严禁只有概述没有细节
   - 所有内容都要详细写出，不能省略任何部分

3. **标记必须包含准确的统计信息（字数、行数、章节等）**
