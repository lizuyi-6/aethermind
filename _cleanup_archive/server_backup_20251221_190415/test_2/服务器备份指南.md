# 服务器备份指南

## 📋 概述

本指南提供了完整的服务器备份和恢复方案，确保您的Flask应用、配置、数据和系统文件都能得到安全备份。

## 🎯 备份内容

备份脚本会自动备份以下内容：

### 1. 应用代码
- Python文件（`*.py`）
- HTML模板（`templates/`）
- 静态资源（`static/`）
- 小程序代码（`miniprogram/`）
- 依赖文件（`requirements.txt`）

### 2. 配置文件
- `config.py` - 应用配置
- `system_prompt.txt` - 系统提示词
- `.env` - 环境变量（如果存在）
- `可行性研究报告模板.md` - 报告模板

### 3. 数据文件
- `generated_codes.json` - 兑换码数据
- 用户数据（如果有）

### 4. 系统配置
- `flask-app.service` - systemd服务配置
- nginx配置文件
- Python环境信息
- 系统信息

### 5. 上传文件
- `uploads/` 目录中的所有文件

### 6. 生成的报告
- 所有生成的 `.md` 报告文件

## 🚀 使用方法

### 方法一：在服务器上直接运行（推荐）

#### 1. 上传备份脚本到服务器

```bash
# 从Windows本地
scp -i "C:\Users\Abraham\Downloads\KeyPair-6418.pem" -P 2950 服务器备份脚本.sh root@60.10.230.156:/var/www/html/
```

#### 2. SSH连接到服务器

```bash
ssh -i "C:\Users\Abraham\Downloads\KeyPair-6418.pem" -p 2950 root@60.10.230.156
```

#### 3. 运行备份脚本

```bash
cd /var/www/html
chmod +x 服务器备份脚本.sh
./服务器备份脚本.sh
```

#### 4. 查看备份结果

备份完成后，会显示：
- 备份文件位置：`/root/backups/backup_YYYYMMDD_HHMMSS.tar.gz`
- 备份文件大小
- 备份内容统计

### 方法二：使用Windows批处理脚本下载备份

#### 1. 在服务器上运行备份（如果还没运行）

```bash
# SSH到服务器
ssh -i "C:\Users\Abraham\Downloads\KeyPair-6418.pem" -p 2950 root@60.10.230.156

# 运行备份
cd /var/www/html
./服务器备份脚本.sh
```

#### 2. 在Windows上运行下载脚本

双击运行 `下载服务器备份.bat`，脚本会自动：
- 连接到服务器
- 查找最新备份
- 下载到本地 `服务器备份` 目录

## 🔄 恢复备份

### 方法一：在同一台服务器上恢复

#### 1. 上传恢复脚本

```bash
scp -i "C:\Users\Abraham\Downloads\KeyPair-6418.pem" -P 2950 服务器备份恢复脚本.sh root@60.10.230.156:/var/www/html/
```

#### 2. SSH到服务器并运行恢复

```bash
ssh -i "C:\Users\Abraham\Downloads\KeyPair-6418.pem" -p 2950 root@60.10.230.156
cd /var/www/html
chmod +x 服务器备份恢复脚本.sh
./服务器备份恢复脚本.sh /root/backups/backup_20241209_120000.tar.gz
```

#### 3. 按照提示操作

恢复脚本会：
- 停止Flask服务
- 备份当前文件（安全措施）
- 解压备份文件
- 恢复应用代码、配置、数据
- 询问是否恢复上传文件和报告
- 询问是否恢复系统配置
- 启动服务

### 方法二：在新服务器上恢复（跨服务器恢复）⭐

这是**推荐的方法**，可以在任何新服务器上完整恢复应用。

#### 方式A：使用Windows批处理脚本（推荐）

1. **上传备份和脚本到新服务器**

   双击运行 `上传备份到新服务器.bat`，脚本会：
   - 提示输入新服务器IP地址
   - 提示选择本地备份文件
   - 自动上传备份文件和恢复脚本

2. **SSH到新服务器并运行恢复**

   ```bash
   ssh -i "你的密钥.pem" -p 端口 root@新服务器IP
   cd /root
   chmod +x 跨服务器恢复脚本.sh
   ./跨服务器恢复脚本.sh /root/backup_20241209_120000.tar.gz
   ```

#### 方式B：手动上传

1. **上传备份文件到新服务器**

   ```bash
   scp -i "你的密钥.pem" -p 端口 backup_20241209_120000.tar.gz root@新服务器IP:/root/
   ```

2. **上传恢复脚本**

   ```bash
   scp -i "你的密钥.pem" -p 端口 跨服务器恢复脚本.sh root@新服务器IP:/root/
   ```

3. **SSH到新服务器并运行恢复**

   ```bash
   ssh -i "你的密钥.pem" -p 端口 root@新服务器IP
   cd /root
   chmod +x 跨服务器恢复脚本.sh
   ./跨服务器恢复脚本.sh /root/backup_20241209_120000.tar.gz
   ```

#### 跨服务器恢复脚本功能

跨服务器恢复脚本会自动：

1. **环境检测**
   - 自动检测Python3路径和版本
   - 检测pip3路径
   - 检测系统信息

2. **应用部署**
   - 创建应用目录结构
   - 恢复所有应用代码
   - 恢复配置和数据文件

3. **依赖安装**
   - 自动安装Python依赖（从requirements.txt）
   - 如果requirements.txt不存在，安装基础依赖

4. **服务配置**
   - 自动创建systemd服务文件
   - 自动适配Python路径
   - 可选择配置nginx

5. **环境变量配置**
   - 交互式配置API密钥和模型参数
   - 支持OpenAI、通义千问、自定义模型

6. **服务启动**
   - 自动启用并启动服务
   - 检查服务状态
   - 显示日志信息

#### 自定义恢复路径

如果需要使用不同的应用目录或服务名称：

```bash
./跨服务器恢复脚本.sh /root/backup.tar.gz /opt/myapp my-flask-app
```

参数说明：
- 第一个参数：备份文件路径
- 第二个参数（可选）：应用目录，默认 `/var/www/html`
- 第三个参数（可选）：服务名称，默认 `flask-app`

### 手动恢复（部分文件）

如果需要只恢复特定文件：

```bash
# 1. 解压备份到临时目录
mkdir -p /tmp/restore
tar -xzf /root/backups/backup_20241209_120000.tar.gz -C /tmp/restore

# 2. 查看备份内容
ls -R /tmp/restore/backup_*/

# 3. 复制需要的文件
cp /tmp/restore/backup_*/app/agent.py /var/www/html/
cp /tmp/restore/backup_*/data/generated_codes.json /var/www/html/

# 4. 清理临时文件
rm -rf /tmp/restore
```

## ⏰ 自动备份（定时任务）

### 设置每天自动备份

```bash
# 1. 编辑crontab
crontab -e

# 2. 添加以下行（每天凌晨2点备份）
0 2 * * * /var/www/html/服务器备份脚本.sh >> /var/log/backup.log 2>&1
```

### 设置每周自动备份

```bash
# 每周日凌晨3点备份
0 3 * * 0 /var/www/html/服务器备份脚本.sh >> /var/log/backup.log 2>&1
```

### 查看备份日志

```bash
tail -f /var/log/backup.log
```

## 📊 备份管理

### 查看所有备份

```bash
ls -lh /root/backups/
```

### 备份文件命名规则

```
backup_YYYYMMDD_HHMMSS.tar.gz
```

例如：`backup_20241209_143022.tar.gz` 表示2024年12月9日14:30:22的备份

### 自动清理旧备份

备份脚本会自动保留最近5个备份，删除更旧的备份。如需修改保留数量，编辑 `服务器备份脚本.sh` 中的以下行：

```bash
# 修改数字5为你想保留的备份数量
if [ "$BACKUP_COUNT" -gt 5 ]; then
```

### 手动删除旧备份

```bash
# 删除特定备份
rm /root/backups/backup_20241201_120000.tar.gz

# 删除7天前的备份
find /root/backups -name "backup_*.tar.gz" -mtime +7 -delete
```

## 🔒 备份安全建议

### 1. 加密备份（可选）

```bash
# 使用gpg加密备份
gpg -c backup_20241209_120000.tar.gz
# 会生成 backup_20241209_120000.tar.gz.gpg
```

### 2. 备份到远程存储

#### 使用rsync备份到另一台服务器

```bash
rsync -avz -e "ssh -i /path/to/key.pem -p 2950" \
  /root/backups/ user@backup-server:/backups/
```

#### 使用scp备份到本地

```bash
# 定期从服务器下载备份
scp -i "key.pem" -P 2950 root@server:/root/backups/*.tar.gz ./local_backups/
```

### 3. 验证备份完整性

```bash
# 测试备份文件是否完整
tar -tzf backup_20241209_120000.tar.gz > /dev/null && echo "备份文件完整" || echo "备份文件损坏"
```

## 📝 备份检查清单

定期检查以下项目：

- [ ] 备份脚本是否正常运行
- [ ] 备份文件是否成功创建
- [ ] 备份文件大小是否合理（不应为0）
- [ ] 备份内容是否完整（检查backup_info.txt）
- [ ] 旧备份是否自动清理
- [ ] 备份文件是否已下载到本地
- [ ] 恢复测试是否成功（可选，定期测试）

## 🆘 故障排除

### 问题1：备份脚本权限不足

```bash
chmod +x 服务器备份脚本.sh
```

### 问题2：磁盘空间不足

```bash
# 检查磁盘空间
df -h

# 清理旧备份
find /root/backups -name "backup_*.tar.gz" -mtime +30 -delete
```

### 问题3：备份文件损坏

```bash
# 测试备份文件
tar -tzf backup_*.tar.gz

# 如果损坏，删除并重新备份
rm backup_*.tar.gz
./服务器备份脚本.sh
```

### 问题4：恢复后服务无法启动

```bash
# 检查服务状态
systemctl status flask-app.service

# 查看详细日志
journalctl -u flask-app.service -n 100

# 检查文件权限
ls -la /var/www/html/*.py

# 检查Python路径
which python3
```

## 📚 相关文件

### 备份相关
- `服务器备份脚本.sh` - 服务器端备份脚本
- `下载服务器备份.bat` - Windows端下载脚本

### 恢复相关
- `服务器备份恢复脚本.sh` - 同服务器恢复脚本（适用于原服务器恢复）

## 💡 最佳实践

1. **定期备份**：建议每天至少备份一次
2. **多重备份**：保留多个时间点的备份
3. **异地备份**：将备份下载到本地或其他服务器
4. **测试恢复**：定期测试备份恢复流程（建议在新服务器上测试）
5. **监控备份**：设置日志监控，确保备份成功
6. **版本控制**：重要代码变更前先备份
7. **跨服务器测试**：定期在新服务器上测试恢复流程，确保备份可移植性
8. **文档记录**：记录API密钥等敏感信息（安全存储），恢复时需要
