## 📝 详细步骤

### 步骤 1：SSH 连接到服务器

```bash
ssh -i "C:\Users\Abraham\Downloads\KeyPair-6418.pem" -p 2950 root@60.10.230.156
```

### 步骤 2：编辑 .env 文件

```bash
cd /var/www/html
nano .env
```

### 步骤 3：配置通义千问

在 `.env` 文件中添加或修改以下内容：

```bash
# 使用通义千问
MODEL_PROVIDER=tongyi
DASHSCOPE_API_KEY=你的通义千问API密钥
MODEL_NAME=qwen3-max

# 可选参数
TEMPERATURE=0.7
MAX_TOKENS=32000
```

**配置说明**：
- `MODEL_PROVIDER=tongyi`：指定使用通义千问
- `DASHSCOPE_API_KEY`：你的通义千问 API 密钥（**必需**）
  - 获取方式：登录阿里云控制台 → 通义千问服务 → API 密钥管理
- `MODEL_NAME=qwen3-max`：默认模型（推荐），也可以使用其他模型
- `API_BASE_URL`：**不需要设置**，系统会自动使用通义千问的官方地址

### 步骤 4：验证配置

```bash
# 检查环境变量
cat .env

# 测试 Python 能否读取配置
python3 -c "from dotenv import load_dotenv; import os; load_dotenv(); print('MODEL_PROVIDER:', os.getenv('MODEL_PROVIDER')); print('DASHSCOPE_API_KEY:', '已设置' if os.getenv('DASHSCOPE_API_KEY') else '未设置')"
```

### 步骤 5：重启 Flask 应用

```bash
# 停止当前运行的 Flask
pkill -f "python3 app.py"

# 重新启动
cd /var/www/html
python3 app.py
```

## 🐛 常见问题

### Q1: 提示 "未设置API密钥"

**原因**：`DASHSCOPE_API_KEY` 未设置或设置错误。

**解决方法**：
```bash
# 检查 .env 文件
cat /var/www/html/.env | grep DASHSCOPE_API_KEY

# 如果为空，重新设置
nano /var/www/html/.env
# 添加：DASHSCOPE_API_KEY=你的API密钥
```

### Q2: 连接超时

**原因**：网络问题或 API 密钥无效。

**解决方法**：
```bash
# 测试 API 连接
curl -X POST https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer 你的API密钥" \
  -d '{
    "model": "qwen3-max",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```

### Q3: 模型不存在错误

**原因**：模型名称错误或该模型不可用。

**解决方法**：
- 检查模型名称是否正确
- 确认你的 API 密钥有权限使用该模型
- 尝试使用 `qwen3-max`（默认模型）

## 🚀 一键切换脚本

在服务器上创建切换脚本：

```bash
cd /var/www/html

# 创建切换到通义千问的脚本
cat > switch_to_tongyi.sh << 'EOF'
#!/bin/bash
cd /var/www/html

# 读取 API 密钥
read -p "请输入通义千问 API 密钥: " api_key

# 创建 .env 文件
cat > .env << EOL
MODEL_PROVIDER=tongyi
DASHSCOPE_API_KEY=${api_key}
MODEL_NAME=qwen3-max
TEMPERATURE=0.7
MAX_TOKENS=32000
EOL

# 重启 Flask
pkill -f "python3 app.py"
echo "配置已更新，正在启动 Flask..."
python3 app.py
EOF

chmod +x switch_to_tongyi.sh
```

使用方法：
```bash
./switch_to_tongyi.sh
```
