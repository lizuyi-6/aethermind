# 微信小程序上线检查清单

在提交审核前，请按照以下清单逐项检查：

## ✅ 基础配置

- [ ] 已注册微信小程序账号
- [ ] 已获取 AppID
- [ ] 已在 `project.config.json` 中配置 AppID
- [ ] 已在 `app.js` 中配置正确的服务器地址（baseUrl）

## ✅ 服务器配置

- [ ] 后端服务器已部署并正常运行
- [ ] 服务器支持 HTTPS（必需）
- [ ] SSL 证书有效且未过期
- [ ] 已配置 CORS，允许微信小程序访问
- [ ] 所有 API 接口可以正常访问

## ✅ 微信公众平台配置

- [ ] 已登录微信公众平台
- [ ] 已配置 request 合法域名（服务器地址）
- [ ] 已配置 uploadFile 合法域名（服务器地址）
- [ ] 已配置 downloadFile 合法域名（如需要）
- [ ] 域名验证已通过

## ✅ 功能测试

- [ ] 聊天功能正常
- [ ] 流式输出正常（模拟流式）
- [ ] 文件上传功能正常
- [ ] 对话历史加载正常
- [ ] 清空对话功能正常
- [ ] 新建对话功能正常
- [ ] 错误提示正常显示

## ✅ 界面检查

- [ ] 首页显示正常
- [ ] 聊天页面显示正常
- [ ] 消息样式正确
- [ ] Markdown 渲染正常
- [ ] 输入框功能正常
- [ ] 按钮交互正常
- [ ] 加载状态显示正常

## ✅ 兼容性测试

- [ ] 在 iOS 设备上测试通过
- [ ] 在 Android 设备上测试通过
- [ ] 不同屏幕尺寸显示正常
- [ ] 横屏/竖屏切换正常

## ✅ 性能检查

- [ ] 页面加载速度正常
- [ ] 消息发送响应及时
- [ ] 文件上传速度正常
- [ ] 无明显的卡顿现象

## ✅ 安全性检查

- [ ] API 密钥未暴露在前端代码中
- [ ] 敏感信息已加密处理
- [ ] 已添加错误处理，不会泄露敏感信息
- [ ] 已添加请求频率限制（后端）

## ✅ 用户体验

- [ ] 操作流程清晰易懂
- [ ] 错误提示友好明确
- [ ] 加载状态有明确提示
- [ ] 界面美观，符合设计规范

## ✅ 代码质量

- [ ] 代码无语法错误
- [ ] 代码已通过微信开发者工具检查
- [ ] 无控制台错误
- [ ] 已移除调试代码和注释

## ✅ 审核准备

- [ ] 已准备功能描述
- [ ] 已选择正确的服务类目
- [ ] 已准备审核截图（如需要）
- [ ] 已阅读微信小程序审核规范

## ✅ 文档准备

- [ ] 已阅读项目 README
- [ ] 已阅读部署指南
- [ ] 已了解常见问题解决方案

## 提交审核前最后检查

1. **在真机上测试**
   - 使用微信扫描开发者工具的预览二维码
   - 在真机上测试所有功能

2. **检查网络请求**
   - 确保所有 API 请求都使用 HTTPS
   - 确保域名已配置

3. **检查错误处理**
   - 测试网络断开情况
   - 测试服务器错误情况
   - 确保有友好的错误提示

4. **检查隐私政策**
   - 如果收集用户信息，需要提供隐私政策
   - 确保符合微信小程序规范

## 提交审核

完成以上所有检查后：

1. 在微信开发者工具中上传代码
2. 填写版本号和项目备注
3. 在微信公众平台提交审核
4. 等待审核结果（通常 1-7 个工作日）

## 审核通过后

1. 在微信公众平台发布小程序
2. 通知用户可以使用
3. 监控运行状态和用户反馈

---

**提示：** 建议先在测试环境充分测试，确认所有功能正常后再提交审核。
