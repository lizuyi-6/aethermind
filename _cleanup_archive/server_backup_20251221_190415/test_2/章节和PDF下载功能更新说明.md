## 🔧 技术实现

### 章节提取逻辑

现在支持三种章节格式的提取：

1. **标准章节格式**（优先）：
   - `第一章 项目概述`
   - `第1章 项目概述`
   - `## 第一章 项目概述`

2. **中文数字格式**：
   - `一、项目概述`
   - `二、项目建设背景`

3. **关键词匹配**：
   - 匹配章节关键词（项目概述、背景、需求等）

### PDF下载流程

1. 用户点击"导出报告"按钮
2. 显示"正在生成PDF..."提示
3. 使用 `html2pdf.js` 生成PDF
4. 自动下载到浏览器默认下载目录
5. 显示下载成功提示，包含：
   - 文件名
   - 保存位置
   - 打开下载文件夹选项

## 📁 PDF文件位置

PDF文件会保存到浏览器的默认下载目录：

- **Windows**：通常为 `C:\Users\用户名\Downloads\`
- **Mac**：通常为 `~/Downloads/`
- **Linux**：通常为 `~/Downloads/`

### 如何查找PDF文件

1. **方法一**：查看浏览器下载记录
   - Chrome/Edge: 按 `Ctrl+J` (Windows) 或 `Cmd+J` (Mac)
   - Firefox: 按 `Ctrl+Shift+Y` (Windows) 或 `Cmd+Shift+Y` (Mac)

2. **方法二**：打开系统下载文件夹
   - Windows: 打开"文件资源管理器"，点击左侧"下载"
   - Mac: 打开"Finder"，点击左侧"下载"
   - Linux: 打开文件管理器，进入"下载"文件夹

3. **方法三**：使用下载成功提示
   - PDF生成后会显示提示框
   - 点击"是"可以尝试打开下载文件夹（部分浏览器支持）

## 🎯 使用说明

### 查看完整章节

1. 生成报告后，报告页面会显示所有10个章节
2. 每个章节都有完整的标题和内容
3. 章节编号从1到10，与报告内容一致

### 下载PDF

1. 在报告页面找到"导出报告"按钮
2. 点击按钮，等待PDF生成（显示"正在生成PDF..."）
3. PDF生成完成后会自动下载
4. 查看下载成功提示，了解文件保存位置
5. 在下载文件夹中找到PDF文件

## ⚠️ 注意事项

1. **PDF文件位置**：
   - PDF文件保存在浏览器的默认下载目录
   - 不是保存在项目目录或服务器上
   - 如果需要保存到特定位置，请修改浏览器下载设置

2. **章节提取**：
   - 系统会自动从报告内容中提取章节
   - 如果提取失败，会使用默认的10个章节
   - 确保报告内容格式正确，以便正确提取章节

3. **浏览器兼容性**：
   - PDF生成功能需要现代浏览器支持
   - 建议使用最新版本的Chrome、Firefox、Safari或Edge

## 🐛 常见问题

### Q: 为什么还是只显示7个章节？

A: 
- 检查报告内容是否包含10个章节
- 如果报告内容只有7个章节，系统会显示实际存在的章节
- 确保报告生成时包含了所有10个章节

### Q: PDF文件在哪里？

A: 
- PDF文件保存在浏览器的默认下载目录
- 查看浏览器下载记录（Chrome/Edge: Ctrl+J）
- 或打开系统的"下载"文件夹

### Q: 如何修改PDF保存位置？

A: 
- 在浏览器设置中修改默认下载位置
- Chrome: 设置 → 高级 → 下载内容 → 位置
- Firefox: 选项 → 常规 → 下载 → 保存文件到

### Q: PDF生成失败怎么办？

A: 
- 检查浏览器是否支持PDF生成
- 确保网络连接正常（首次使用需要加载PDF库）
- 尝试刷新页面后重试
- 检查浏览器控制台是否有错误信息
