# 服务器后台运行快速参考

## 🚀 快速开始

### 方法一：systemd服务（最推荐）⭐

```bash
# 1. 上传文件到服务器后，运行安装脚本
sudo ./install_service.sh

# 2. 或手动安装
sudo cp flask-app.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable flask-app.service
sudo systemctl start flask-app.service

# 3. 检查状态
sudo systemctl status flask-app.service
```

### 方法二：nohup（简单快速）

```bash
# 运行脚本
./start_flask_nohup.sh

# 或手动运行
cd /var/www/html
nohup /usr/local/python3.11/bin/python3 app.py > flask_app.log 2>&1 &
```

### 方法三：screen（可重新连接）

```bash
# 运行脚本
./start_flask_screen.sh

# 或手动运行
screen -dmS flask_app /usr/local/python3.11/bin/python3 /var/www/html/app.py
screen -r flask_app  # 连接查看
```

## 📋 常用命令

### systemd服务管理

```bash
# 查看状态
sudo systemctl status flask-app.service

# 启动/停止/重启
sudo systemctl start flask-app.service
sudo systemctl stop flask-app.service
sudo systemctl restart flask-app.service

# 查看日志
sudo journalctl -u flask-app.service -f

# 启用/禁用开机自启
sudo systemctl enable flask-app.service
sudo systemctl disable flask-app.service
```

### 检查服务状态

```bash
# 运行检查脚本
./check_service.sh

# 或手动检查
ps aux | grep "python3.*app.py"  # 检查进程
netstat -tlnp | grep 5000        # 检查端口
curl http://localhost:5000       # 测试访问
```

## ⚙️ 配置说明

### 修改systemd服务配置

编辑 `flask-app.service`，修改以下内容：

1. **项目路径**: `WorkingDirectory=/var/www/html`
2. **Python路径**: `ExecStart=/usr/local/python3.11/bin/python3`
3. **运行用户**: `User=root`
4. **环境变量**: 在 `[Service]` 部分添加 `Environment="KEY=value"`

### 环境变量配置

在服务文件中添加：

```ini
[Service]
Environment="MODEL_PROVIDER=custom"
Environment="API_BASE_URL=http://60.10.230.156:1025/v1"
Environment="MODEL_NAME=qwen3-32b"
Environment="OPENAI_API_KEY=your-key-here"
```

修改后需要：

```bash
sudo systemctl daemon-reload
sudo systemctl restart flask-app.service
```

## 🔍 故障排查

### 服务无法启动

```bash
# 1. 查看详细日志
sudo journalctl -u flask-app.service -n 50

# 2. 检查Python路径
which python3
/usr/local/python3.11/bin/python3 --version

# 3. 检查文件权限
ls -la /var/www/html/app.py

# 4. 手动测试运行
cd /var/www/html
/usr/local/python3.11/bin/python3 app.py
```

### 端口被占用

```bash
# 查找占用进程
lsof -i :5000
netstat -tlnp | grep 5000

# 停止占用进程
kill <PID>
```

### 权限问题

```bash
# 修改文件所有者
sudo chown -R www-data:www-data /var/www/html/

# 修改权限
sudo chmod -R 755 /var/www/html/
```
