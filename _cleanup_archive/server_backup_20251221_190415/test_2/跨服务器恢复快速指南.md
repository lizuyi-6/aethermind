# 跨服务器恢复快速指南

## 🎯 概述

本指南帮助您将备份从一台服务器恢复到另一台全新的服务器上。

## ⚡ 快速开始（3步完成）

### 步骤1：准备备份文件

在**原服务器**上创建备份：

```bash
# SSH到原服务器
ssh -i "密钥.pem" -p 端口 root@原服务器IP

# 运行备份
cd /var/www/html
./服务器备份脚本.sh
```

备份文件位置：`/root/backups/backup_YYYYMMDD_HHMMSS.tar.gz`

### 步骤2：上传到新服务器

**方式A：使用Windows脚本（推荐）**

双击运行 `上传备份到新服务器.bat`，按提示输入：
- 新服务器IP地址
- 备份文件路径（本地或服务器路径）

**方式B：手动上传**

```bash
# 从本地Windows上传
scp -i "密钥.pem" -p 端口 backup_20241209_120000.tar.gz root@新服务器IP:/root/

# 上传恢复脚本
scp -i "密钥.pem" -p 端口 跨服务器恢复脚本.sh root@新服务器IP:/root/
```

### 步骤3：在新服务器上恢复

```bash
# SSH到新服务器
ssh -i "密钥.pem" -p 端口 root@新服务器IP

# 运行恢复脚本
cd /root
chmod +x 跨服务器恢复脚本.sh
./跨服务器恢复脚本.sh /root/backup_20241209_120000.tar.gz
```

## 📋 恢复过程说明

恢复脚本会引导您完成以下步骤：

### 1. 环境检测 ✅
- 自动检测Python3路径
- 检测pip3
- 显示系统信息

### 2. 应用部署 ✅
- 创建应用目录
- 恢复所有代码文件
- 恢复配置和数据

### 3. 依赖安装 ✅
- 自动安装Python依赖
- 从requirements.txt安装

### 4. 服务配置 ✅
- 创建systemd服务文件
- 自动适配Python路径
- 可选择配置nginx

### 5. 环境变量配置 ⚙️
脚本会询问是否配置环境变量，您需要提供：
- **模型提供商**：openai / tongyi / custom
- **API密钥**：根据提供商选择
  - 通义千问：`DASHSCOPE_API_KEY`
  - OpenAI：`OPENAI_API_KEY`
  - 自定义：`CUSTOM_API_KEY` 和 `API_BASE_URL`
- **模型名称**：如 `qwen3-max`、`gpt-3.5-turbo` 等

### 6. 服务启动 ✅
- 自动启用服务
- 自动启动服务
- 显示服务状态

## 🔧 自定义配置

### 使用不同的应用目录

```bash
./跨服务器恢复脚本.sh /root/backup.tar.gz /opt/myapp
```

### 使用不同的服务名称

```bash
./跨服务器恢复脚本.sh /root/backup.tar.gz /var/www/html my-flask-service
```

### 完整自定义

```bash
./跨服务器恢复脚本.sh <备份文件> <应用目录> <服务名称>
```

## ✅ 恢复后检查清单

恢复完成后，请检查：

- [ ] 服务是否正常运行
  ```bash
  systemctl status flask-app.service
  ```

- [ ] 应用是否响应
  ```bash
  curl http://localhost:5000/
  ```

- [ ] 查看日志是否有错误
  ```bash
  journalctl -u flask-app.service -n 50
  ```

- [ ] 环境变量是否正确
  ```bash
  systemctl show flask-app.service | grep Environment
  ```

- [ ] nginx配置（如果使用）
  ```bash
  nginx -t
  systemctl status nginx
  ```

## 🆘 常见问题

### 问题1：服务启动失败

**检查日志：**
```bash
journalctl -u flask-app.service -n 100
```

**常见原因：**
- Python依赖未安装完整 → 运行 `pip3 install -r /var/www/html/requirements.txt`
- 环境变量未配置 → 检查服务文件中的Environment变量
- 端口被占用 → 检查 `netstat -tlnp | grep 5000`

### 问题2：API调用失败

**检查环境变量：**
```bash
systemctl show flask-app.service | grep -E "MODEL_PROVIDER|API_KEY"
```

**重新配置：**
编辑服务文件 `/etc/systemd/system/flask-app.service`，添加环境变量后：
```bash
systemctl daemon-reload
systemctl restart flask-app.service
```

### 问题3：nginx无法访问

**检查nginx配置：**
```bash
nginx -t
cat /etc/nginx/sites-available/default
```

**检查Flask是否在运行：**
```bash
curl http://127.0.0.1:5000/
```

### 问题4：文件权限问题

```bash
chown -R root:root /var/www/html
chmod +x /var/www/html/*.py
chmod 755 /var/www/html
```

## 📝 恢复后需要手动配置的内容

以下内容需要根据新服务器环境手动配置：

1. **域名和SSL证书**（如果使用HTTPS）
   - 更新nginx配置中的server_name
   - 配置SSL证书路径

2. **防火墙规则**
   ```bash
   # 允许80和443端口
   ufw allow 80/tcp
   ufw allow 443/tcp
   ```

3. **管理员密码**（如果需要修改）
   - 编辑app.py或通过环境变量设置

4. **数据库连接**（如果使用数据库）
   - 配置数据库连接信息
