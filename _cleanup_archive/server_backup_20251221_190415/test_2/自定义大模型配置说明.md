# 大模型配置说明

## 概述

系统支持多种大模型，您可以根据需要选择使用：

### 1. OpenAI（默认，之前使用的大模型）
- **提供商**: OpenAI
- **默认模型**: `gpt-3.5-turbo`
- **需要设置**: `OPENAI_API_KEY` 环境变量

### 2. 通义千问
- **提供商**: 阿里云通义千问
- **默认模型**: `qwen3-max`
- **需要设置**: `DASHSCOPE_API_KEY` 环境变量

### 3. 自定义大模型（您的新模型）
- **API 地址**: `http://60.10.230.156:1025/v1`
- **模型名称**: `qwen3-vl-32b-thinking`
- **API 格式**: OpenAI 兼容格式

## 启动方式

### 方法一：使用统一启动脚本（推荐）⭐

#### 命令行模式
双击运行 **`start.bat`**，然后：
1. 选择要使用的大模型（1-OpenAI, 2-通义千问, 3-自定义大模型）
2. 选择启动模式（1-命令行模式, 2-Web界面模式）

#### Web 界面模式
双击运行 **`start_web.bat`**，然后：
1. 选择要使用的大模型（1-OpenAI, 2-通义千问, 3-自定义大模型）
2. 自动启动Web服务，访问 `http://localhost:5000`

**统一启动脚本的优势：**
- ✅ 一个脚本管理所有大模型
- ✅ 交互式菜单，操作简单
- ✅ 自动检查环境和依赖
- ✅ 启动后可以返回菜单重新选择

### 方法二：使用专用启动脚本（快速启动）

如果您经常使用某个特定的大模型，可以直接使用专用脚本：

#### OpenAI（之前使用的大模型）

**命令行模式：**
- `start_agent.bat` - 原有脚本（默认使用 OpenAI）
- `start_openai.bat` - 明确使用 OpenAI

**Web 界面模式：**
- `start_web_openai.bat` - 明确使用 OpenAI

#### 通义千问

**命令行模式：**
- `start_tongyi.bat`

**Web 界面模式：**
- `start_web_tongyi.bat`

#### 自定义大模型（您的新模型）

**命令行模式：**
- `start_custom_model.bat`

**Web 界面模式：**
- `start_web_custom.bat`

### 方法二：手动设置环境变量

#### 使用 OpenAI（之前的大模型）

**Windows (CMD):**
```cmd
set MODEL_PROVIDER=openai
set OPENAI_API_KEY=your-api-key-here
python agent.py
```

#### 使用通义千问

**Windows (CMD):**
```cmd
set MODEL_PROVIDER=tongyi
set DASHSCOPE_API_KEY=your-api-key-here
python agent.py
```

#### 使用自定义大模型

**Windows (CMD):**
```cmd
set MODEL_PROVIDER=custom
set API_BASE_URL=http://60.10.230.156:1025/v1
set MODEL_NAME=qwen3-vl-32b-thinking
python agent.py
```

### 方法三：创建 .env 文件

在项目根目录创建 `.env` 文件，根据要使用的大模型设置相应内容：

**使用 OpenAI：**
```
MODEL_PROVIDER=openai
OPENAI_API_KEY=your-api-key-here
```

**使用通义千问：**
```
MODEL_PROVIDER=tongyi
DASHSCOPE_API_KEY=your-api-key-here
```

**使用自定义大模型：**
```
MODEL_PROVIDER=custom
API_BASE_URL=http://60.10.230.156:1025/v1
MODEL_NAME=qwen3-vl-32b-thinking
```

然后正常启动程序即可。

## 配置说明

### 默认配置

系统已自动配置以下默认值（在 `config.py` 中）：

- **默认提供商**: `openai`（之前使用的大模型）
- 当 `MODEL_PROVIDER=custom` 时：
  - 默认 `API_BASE_URL`: `http://60.10.230.156:1025/v1`
  - 默认 `MODEL_NAME`: `qwen3-vl-32b-thinking`

### API 密钥

如果您的 API 需要密钥，可以通过以下方式设置：

**环境变量：**
```bash
set CUSTOM_API_KEY=your-api-key-here
```

**或 .env 文件：**
```
CUSTOM_API_KEY=your-api-key-here
```

如果 API 不需要密钥，可以留空或不设置。

## 验证配置

启动程序后，系统会显示当前使用的配置信息。您也可以查看程序输出的配置信息来确认是否正确。

## 测试连接

启动程序后，可以尝试发送一条测试消息，例如：
```
你好，你是谁？
```

如果配置正确，应该能收到大模型的回复。

## 故障排除

### 问题：连接失败

**可能原因：**
1. API 地址不正确
2. 网络连接问题
3. API 服务未启动

**解决方法：**
1. 检查 API 地址是否正确：`http://60.10.230.156:1025/v1`
2. 使用浏览器或 curl 测试 API 是否可访问
3. 检查防火墙设置

### 问题：模型名称错误

**解决方法：**
确保 `MODEL_NAME` 设置为 `qwen3-vl-32b-thinking`

### 问题：API 密钥错误

**解决方法：**
如果 API 需要密钥，请设置 `CUSTOM_API_KEY` 环境变量

## 其他配置选项

您还可以通过环境变量配置其他参数：

- `TEMPERATURE`: 温度参数（0-2），默认 0.7
- `MAX_TOKENS`: 最大 token 数，默认 32000

示例：
```bash
set TEMPERATURE=0.8
set MAX_TOKENS=40000
```

## 技术支持

如有问题，请检查：
1. Python 版本（需要 3.7+）
2. 依赖包是否已安装（运行 `pip install -r requirements.txt`）
3. 网络连接是否正常
4. API 服务是否正常运行
