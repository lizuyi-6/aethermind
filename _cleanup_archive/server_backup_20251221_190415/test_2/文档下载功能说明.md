# 文档下载功能说明

## 功能概述

现在服务器端生成的可行性研究报告可以通过网页直接下载到用户本地，无需手动访问服务器文件系统。

## 功能特点

- ✅ **自动保存**：报告生成后自动保存到服务器的 `reports/` 目录
- ✅ **网页下载**：用户可以通过点击下载链接直接在浏览器中下载
- ✅ **安全下载**：使用安全的文件路径检查，防止路径遍历攻击
- ✅ **流式和非流式支持**：支持流式输出和非流式输出两种模式

## 工作流程

### 1. 报告生成
当用户请求生成可行性研究报告时：
- 系统生成完整的报告内容
- 自动保存到服务器的 `reports/` 目录
- 生成唯一的文件名（包含项目名称和时间戳）

### 2. 下载链接生成
- 系统自动生成下载URL：`/api/download/report/{filename}`
- 在回复消息中显示下载链接
- 流式输出中也会包含下载信息

### 3. 用户下载
- 用户点击下载链接
- 浏览器自动下载Markdown格式的报告文件
- 文件保存到用户的默认下载目录

## API接口

### 下载报告接口

**URL**: `/api/download/report/<filename>`

**方法**: `GET`

**参数**:
- `filename`: 报告文件名（由系统生成）

**响应**:
- 成功：返回Markdown文件，浏览器自动下载
- 失败：返回JSON错误信息

**示例**:
```
GET /api/download/report/新能源项目_可行性研究报告_20241215_143022.md
```

## 文件存储

### 存储位置
- 目录：`reports/`
- 格式：Markdown (.md)
- 命名规则：`{项目名称}_可行性研究报告_{时间戳}.md`

### 文件命名示例
```
新能源项目_可行性研究报告_20241215_143022.md
智能制造项目_可行性研究报告_20241215_150530.md
```

## 安全措施

1. **路径安全检查**：只允许下载 `reports/` 目录下的文件
2. **文件名验证**：使用 `secure_filename` 确保文件名安全
3. **路径遍历防护**：检查文件路径，防止 `../` 等攻击

## 前端集成

### 流式输出模式

在流式输出中，系统会发送包含下载信息的消息：

```json
{
  "content": "✅ **报告已生成并保存**\n📁 文件名: `xxx.md`\n🔗 [点击下载报告](/api/download/report/xxx.md)",
  "done": false,
  "report_download": {
    "url": "/api/download/report/xxx.md",
    "filename": "xxx.md"
  }
}
```

### 非流式输出模式

在非流式输出中，响应包含：

```json
{
  "reply": "...",
  "session_id": "...",
  "usage": {...},
  "report_download": {
    "url": "/api/download/report/xxx.md",
    "filename": "xxx.md"
  }
}
```

```

🔗 [点击下载报告](/api/download/report/新能源项目_可行性研究报告_20241215_143022.md)
```

1. 点击下载链接
2. 浏览器自动下载文件

## 注意事项

1. **文件保留**：报告文件会保留在服务器上，建议定期清理旧文件
2. **存储空间**：确保服务器有足够的存储空间
3. **文件权限**：确保 `reports/` 目录有写入权限
4. **并发下载**：支持多个用户同时下载不同的报告文件
