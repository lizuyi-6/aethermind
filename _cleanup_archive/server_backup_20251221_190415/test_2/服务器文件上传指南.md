# 服务器文件上传指南

## 🔴 问题：找不到 app.py 文件

错误信息：
```
python3: can't open file '/var/www/html/app.py': [Errno 2] No such file or directory
```

**原因**：项目文件还没有上传到服务器。

---

## 📋 需要上传的文件

### 核心程序文件（必需）
- `app.py` - Flask Web应用主程序
- `agent.py` - 智能体程序
- `config.py` - 配置文件
- `file_processor.py` - 文件处理模块
- `requirements.txt` - Python依赖

### 前端文件（必需）
- `templates/` 目录（包含 HTML 模板）
- `static/` 目录（包含 CSS、JS 文件）
- `index_static.html` - 静态HTML文件（可选）

### 配置文件（可选）
- `system_prompt.txt` - 系统提示词
- `system_prompt_enhanced.txt` - 增强提示词

### 目录（需要创建）
- `uploads/` - 文件上传目录（在服务器上创建）

---

## 🚀 上传方法

### 方法一：使用 SCP 批量上传（推荐）

在**本地电脑**（Windows PowerShell）执行：

```powershell
# 进入项目目录
cd C:\Users\Abraham\Desktop\挑战杯\test--1\test_2

# 设置密钥文件路径
$KEY_FILE = "C:\Users\Abraham\Downloads\KeyPair-6418.pem"
$PORT = 2950
$SERVER = "root@60.10.230.156"
$REMOTE_DIR = "/var/www/html"

# 上传核心文件
scp -i $KEY_FILE -P $PORT app.py ${SERVER}:${REMOTE_DIR}/
scp -i $KEY_FILE -P $PORT agent.py ${SERVER}:${REMOTE_DIR}/
scp -i $KEY_FILE -P $PORT config.py ${SERVER}:${REMOTE_DIR}/
scp -i $KEY_FILE -P $PORT file_processor.py ${SERVER}:${REMOTE_DIR}/
scp -i $KEY_FILE -P $PORT requirements.txt ${SERVER}:${REMOTE_DIR}/

# 上传 templates 目录
scp -i $KEY_FILE -P $PORT -r templates ${SERVER}:${REMOTE_DIR}/

# 上传 static 目录
scp -i $KEY_FILE -P $PORT -r static ${SERVER}:${REMOTE_DIR}/

# 上传配置文件（可选）
scp -i $KEY_FILE -P $PORT system_prompt.txt ${SERVER}:${REMOTE_DIR}/
scp -i $KEY_FILE -P $PORT system_prompt_enhanced.txt ${SERVER}:${REMOTE_DIR}/
```

### 方法二：使用 FTP 工具上传

1. 下载 FileZilla：https://filezilla-project.org/
2. 连接到服务器：
   - 主机：`60.10.230.156`
   - 用户名：`root`
   - 密码：您的服务器密码
   - 端口：`2950`（SFTP）或 `21`（FTP）
3. 上传文件到 `/var/www/html/` 目录

## 📝 服务器端设置

上传文件后，在服务器上执行：

### 步骤 1：进入项目目录

```bash
cd /var/www/html
ls -la  # 查看文件是否上传成功
```

### 步骤 2：创建必要的目录

```bash
# 创建 uploads 目录
mkdir -p uploads
chmod 755 uploads
```

### 步骤 3：设置文件权限

```bash
# 设置 Python 文件权限
chmod 644 *.py
chmod 644 *.txt

# 设置目录权限
chmod -R 755 templates/
chmod -R 755 static/
```

### 步骤 4：安装依赖

```bash
# 如果还没有 requirements.txt，先创建
# （参考之前的指南）

# 安装依赖
python3 -m pip install --upgrade pip
python3 -m pip install -r requirements.txt
```

### 步骤 5：测试运行

```bash
# 测试 Flask 应用
python3 app.py
```

如果看到类似以下输出，说明运行成功：
```
智能体Web服务已启动
访问地址: http://localhost:5000
```

## 🔧 快速部署脚本

### 在服务器上创建部署脚本

```bash
cd /var/www/html

# 创建部署脚本
cat > deploy.sh << 'EOF'
#!/bin/bash
echo "开始部署..."

# 创建 uploads 目录
mkdir -p uploads
chmod 755 uploads

# 设置文件权限
chmod 644 *.py *.txt
chmod -R 755 templates/ static/

# 安装依赖
echo "安装依赖..."
python3 -m pip install --upgrade pip
python3 -m pip install -r requirements.txt

echo "部署完成！"
echo "运行: python3 app.py"
EOF

# 设置执行权限
chmod +x deploy.sh

# 运行部署脚本
./deploy.sh
```

## 🎯 完整的部署流程

### 1. 在本地准备文件

确保以下文件存在：
- `app.py`
- `agent.py`
- `config.py`
- `file_processor.py`

- `static/` 目录

```bash
cd /var/www/html
```

```bash
python3 app.py
```

```bash

python3 app.py
```

## ⚠️ 注意事项

1. **确保在正确的目录**：`/var/www/html/`
2. **检查文件权限**：Python 文件需要可读权限
3. **检查端口**：确保 5000 端口没有被占用
4. **防火墙设置**：确保防火墙允许 5000 端口访问

---

## 🐛 常见问题

### Q: 上传后文件权限不对？

A: 执行 `chmod 644 *.py` 和 `chmod -R 755 templates/ static/`

### Q: 找不到模块？

A: 检查依赖是否安装：`python3 -m pip list`

### Q: 端口被占用？

A: 检查端口：`netstat -tlnp | grep 5000` 或修改 `app.py` 中的端口
