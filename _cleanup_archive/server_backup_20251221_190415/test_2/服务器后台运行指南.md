# 服务器后台运行指南

## 概述

本指南提供多种方法确保Flask服务在SSH连接断开后继续运行。推荐按优先级选择：

1. **systemd服务**（最推荐）⭐ - 系统级服务管理，自动重启，开机自启
2. **nohup方式**（简单快速） - 适合临时运行
3. **screen方式**（可重新连接） - 适合需要查看输出的场景

---

## 方法一：使用systemd服务（最推荐）⭐

### 优点
- ✅ SSH断开后继续运行
- ✅ 系统崩溃后自动重启
- ✅ 开机自动启动
- ✅ 统一日志管理
- ✅ 服务状态管理方便

### 配置步骤

#### 1. 检查现有服务文件

项目已包含 `flask-app.service` 文件，需要根据实际情况修改：

```bash
# 查看当前服务文件
cat flask-app.service
```

#### 2. 修改服务文件

根据你的实际路径修改 `flask-app.service`：

```bash
# 编辑服务文件
nano flask-app.service
```

**需要修改的配置项：**
- `WorkingDirectory`: 项目实际路径（如：`/var/www/html` 或 `/home/username/project`）
- `ExecStart`: Python解释器路径和app.py路径
- `User`: 运行服务的用户（建议使用非root用户）
- `Environment`: 环境变量（API密钥等）

**示例配置：**
```ini
[Unit]
Description=Flask Web Application - 超智引擎
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/html
Environment="PATH=/usr/local/bin:/usr/bin:/bin:/usr/local/python3.11/bin"
Environment="MODEL_PROVIDER=custom"
Environment="API_BASE_URL=http://60.10.230.156:1025/v1"
Environment="MODEL_NAME=qwen3-32b"
Environment="OPENAI_API_KEY=your-key-here"
ExecStart=/usr/local/python3.11/bin/python3 /var/www/html/app.py
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

#### 3. 安装服务

```bash
# 复制服务文件到systemd目录
sudo cp flask-app.service /etc/systemd/system/

# 重新加载systemd配置
sudo systemctl daemon-reload

# 启用服务（开机自启）
sudo systemctl enable flask-app.service

# 启动服务
sudo systemctl start flask-app.service
```

#### 4. 管理服务

```bash
# 查看服务状态
sudo systemctl status flask-app.service

# 查看服务日志
sudo journalctl -u flask-app.service -f

# 停止服务
sudo systemctl stop flask-app.service

# 重启服务
sudo systemctl restart flask-app.service

# 禁用开机自启
sudo systemctl disable flask-app.service
```

#### 5. 验证服务运行

```bash
# 检查服务是否运行
sudo systemctl is-active flask-app.service

# 检查端口是否监听
netstat -tlnp | grep 5000
# 或
ss -tlnp | grep 5000

# 测试访问
curl http://localhost:5000
```

## 方法二：使用nohup（简单快速）

### 优点
- ✅ 配置简单
- ✅ SSH断开后继续运行
- ✅ 日志输出到文件

### 缺点
- ❌ 不会自动重启
- ❌ 需要手动管理进程

### 使用步骤

#### 1. 使用提供的脚本

项目已包含 `start_flask_nohup.sh` 脚本：

```bash
# 赋予执行权限
chmod +x start_flask_nohup.sh

# 运行脚本
./start_flask_nohup.sh
```

#### 2. 手动使用nohup

```bash
# 进入项目目录
cd /var/www/html

# 停止可能正在运行的进程
pkill -f "python3.*app.py" || true

# 使用nohup启动（后台运行）
nohup /usr/local/python3.11/bin/python3 app.py > flask_app.log 2>&1 &

# 查看进程ID
echo $!
```

#### 3. 管理服务

```bash
# 查看日志
tail -f /var/www/html/flask_app.log

# 查看进程
ps aux | grep "python3.*app.py"

# 停止服务
pkill -f "python3.*app.py"

# 或使用进程ID
kill <PID>
```

## 方法三：使用screen（可重新连接）

### 优点
- ✅ SSH断开后可重新连接查看输出
- ✅ 可以交互式操作
- ✅ 适合调试场景

### 缺点
- ❌ 需要安装screen
- ❌ 不会自动重启

### 使用步骤

#### 1. 安装screen（如未安装）

```bash
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install screen

# CentOS/RHEL
sudo yum install screen
```

#### 2. 使用提供的脚本

项目已包含 `start_flask_screen.sh` 脚本：

```bash
# 赋予执行权限
chmod +x start_flask_screen.sh

# 运行脚本
./start_flask_screen.sh
```

#### 3. 手动使用screen

```bash
# 创建screen会话并启动服务
screen -dmS flask_app /usr/local/python3.11/bin/python3 /var/www/html/app.py

# 查看所有screen会话
screen -list

# 连接到会话（查看输出）
screen -r flask_app

# 断开会话（服务继续运行）
# 在screen中按: Ctrl+A 然后按 D

# 停止服务
screen -S flask_app -X quit
```

## 方法四：使用tmux（screen的替代）

### 优点
- ✅ 功能比screen更强大
- ✅ 支持分屏
- ✅ SSH断开后可重新连接

### 使用步骤

```bash
# 安装tmux
sudo apt-get install tmux  # Ubuntu/Debian
sudo yum install tmux      # CentOS/RHEL

# 创建tmux会话并启动服务
tmux new-session -d -s flask_app 'cd /var/www/html && /usr/local/python3.11/bin/python3 app.py'

# 查看会话
tmux list-sessions

# 连接到会话
tmux attach-session -t flask_app

# 断开会话（服务继续运行）
# 在tmux中按: Ctrl+B 然后按 D

# 停止服务
tmux kill-session -t flask_app
```

## 快速检查清单

### 服务是否在运行？

```bash
# 方法1: 检查systemd服务
sudo systemctl status flask-app.service

# 方法2: 检查进程
ps aux | grep "python3.*app.py"

# 方法3: 检查端口
netstat -tlnp | grep 5000
ss -tlnp | grep 5000

# 方法4: 测试HTTP访问
curl http://localhost:5000
curl http://your-server-ip:5000
```

### 服务无法启动的排查

```bash
# 1. 检查Python路径是否正确
which python3
/usr/local/python3.11/bin/python3 --version

# 2. 检查app.py是否存在
ls -la /var/www/html/app.py

# 3. 检查依赖是否安装
/usr/local/python3.11/bin/python3 -c "import flask"

# 4. 检查端口是否被占用
lsof -i :5000
netstat -tlnp | grep 5000

# 5. 检查权限
ls -la /var/www/html/
sudo chown -R www-data:www-data /var/www/html/

# 6. 查看详细错误日志
# systemd方式:
sudo journalctl -u flask-app.service -n 50

# nohup方式:
tail -n 50 /var/www/html/flask_app.log

# screen方式:
screen -r flask_app
```

## 推荐配置方案

### 生产环境（推荐）

使用 **systemd服务**：
- 自动重启
- 开机自启
- 统一日志管理
- 系统级管理

### 开发/测试环境

使用 **screen或tmux**：
- 方便查看实时输出
- 可以交互式调试
- 快速重启

### 临时运行

使用 **nohup**：
- 快速启动
- 简单配置

## 环境变量配置

如果使用systemd服务，需要在服务文件中配置环境变量：

```ini
[Service]
Environment="MODEL_PROVIDER=custom"
Environment="API_BASE_URL=http://60.10.230.156:1025/v1"
Environment="MODEL_NAME=qwen3-32b"
Environment="OPENAI_API_KEY=your-key-here"
Environment="DASHSCOPE_API_KEY=your-key-here"
```

或者创建环境文件：

```bash
# 创建环境文件
sudo nano /etc/flask-app.env

# 内容：
MODEL_PROVIDER=custom
API_BASE_URL=http://60.10.230.156:1025/v1
MODEL_NAME=qwen3-32b
OPENAI_API_KEY=your-key-here

# 在服务文件中引用
[Service]
EnvironmentFile=/etc/flask-app.env
```

## 防火墙配置

确保服务器防火墙允许5000端口：

```bash
# Ubuntu/Debian (ufw)
sudo ufw allow 5000/tcp
sudo ufw reload

# CentOS/RHEL (firewalld)
sudo firewall-cmd --permanent --add-port=5000/tcp
sudo firewall-cmd --reload

# 或直接关闭防火墙（不推荐生产环境）
sudo systemctl stop firewalld
```

## 常见问题

### Q1: 服务启动后立即停止？

**检查：**
- Python路径是否正确
- app.py文件路径是否正确
- 查看日志：`sudo journalctl -u flask-app.service -n 50`

### Q2: 端口5000已被占用？

**解决：**
```bash
# 查找占用端口的进程
lsof -i :5000
# 或
netstat -tlnp | grep 5000

# 停止占用进程
kill <PID>

# 或修改app.py中的端口号
```

### Q3: 权限不足？

**解决：**
```bash
# 检查文件权限
ls -la /var/www/html/

# 修改所有者
sudo chown -R www-data:www-data /var/www/html/

# 修改权限
sudo chmod -R 755 /var/www/html/
```

### Q4: 环境变量未生效？

**解决：**
- 在systemd服务文件中添加Environment行
- 或创建环境文件并在服务文件中引用
- 重启服务：`sudo systemctl restart flask-app.service`

## 总结

**最佳实践：**
1. 生产环境使用 **systemd服务**
2. 开发环境使用 **screen/tmux**
3. 临时测试使用 **nohup**

**记住：**
- 使用systemd后，服务会在SSH断开后继续运行
- 服务会在系统重启后自动启动（如果启用了enable）
- 服务崩溃后会自动重启（Restart=always）
