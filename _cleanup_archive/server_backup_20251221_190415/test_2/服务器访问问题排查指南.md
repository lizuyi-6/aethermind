# 服务器访问问题排查指南

## 🔴 问题：Nginx运行正常但无法访问

Nginx服务正在运行，但无法访问网站。

---

## 📋 排查步骤

### 步骤 1：检查 Flask 应用是否运行

```bash
# 检查 Flask 进程
ps aux | grep python
ps aux | grep app.py

# 检查端口 5000 是否被监听
netstat -tlnp | grep 5000
# 或
ss -tlnp | grep 5000
```

**如果 Flask 没有运行**：
```bash
cd /var/www/html
python3 app.py
```

### 步骤 2：检查 Nginx 配置

```bash
# 查看 Nginx 配置
sudo nginx -t

# 查看 Nginx 配置文件
cat /etc/nginx/sites-available/default
# 或
cat /etc/nginx/sites-enabled/default
```

### 步骤 3：检查防火墙

```bash
# 检查防火墙状态（Ubuntu/Debian）
sudo ufw status

# 检查防火墙状态（CentOS/RHEL）
sudo firewall-cmd --list-all

# 开放端口（如果需要）
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 5000/tcp  # Flask端口
```

### 步骤 4：检查端口监听

```bash
# 检查所有监听的端口
netstat -tlnp
# 或
ss -tlnp

# 应该看到：
# :80 (HTTP)
# :443 (HTTPS，如果配置了SSL)
# :5000 (Flask，如果直接访问)
```

### 步骤 5：检查域名DNS解析

```bash
# 检查域名解析
nslookup chaozhiyinqin.xyz
# 或
dig chaozhiyinqin.xyz

# 应该显示：60.10.230.156
```

### 步骤 6：测试本地访问

```bash
# 在服务器上测试本地访问
curl http://localhost:5000
curl http://localhost:80
```

## 🔧 常见问题及解决方案

### 问题 1：Flask 应用没有运行

**解决方案**：
```bash
cd /var/www/html
python3 app.py
```

**后台运行**：
```bash
# 使用 nohup
nohup python3 app.py > app.log 2>&1 &

# 或使用 screen
screen -S flask
python3 app.py
# 按 Ctrl+A, D 退出（服务继续运行）
```

### 问题 2：Nginx 配置不正确

**检查配置文件**：
```bash
sudo nano /etc/nginx/sites-available/default
```

**正确的配置应该包含**：
```nginx
server {
    listen 80;
    server_name chaozhiyinqin.xyz;

    location / {
        proxy_pass http://localhost:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

**应用配置**：
```bash
sudo nginx -t
sudo systemctl reload nginx
```

### 问题 3：防火墙阻止访问

**Ubuntu/Debian**：
```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw reload
```

**CentOS/RHEL**：
```bash
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

### 问题 4：端口被占用

```bash
# 检查端口占用
sudo lsof -i :80
sudo lsof -i :5000

# 如果端口被占用，可以：
# 1. 停止占用端口的服务
# 2. 或修改 Flask/Nginx 的端口配置
```

### 问题 5：SELinux 阻止（CentOS/RHEL）

```bash
# 检查 SELinux 状态
getenforce

# 如果显示 Enforcing，需要设置
sudo setsebool -P httpd_can_network_connect 1
```

## 🚀 快速诊断命令

在服务器上执行以下命令进行快速诊断：

```bash
echo "=== 1. 检查 Flask 进程 ==="
ps aux | grep python | grep app.py

echo "=== 2. 检查端口监听 ==="
netstat -tlnp | grep -E ':(80|443|5000)'

echo "=== 3. 检查 Nginx 状态 ==="
sudo systemctl status nginx

echo "=== 4. 检查防火墙 ==="
sudo ufw status 2>/dev/null || sudo firewall-cmd --list-all 2>/dev/null

echo "=== 5. 测试本地访问 ==="
curl -I http://localhost:5000
curl -I http://localhost:80

echo "=== 6. 检查 Nginx 配置 ==="
sudo nginx -t
```

## 📝 完整的 Nginx 配置示例

创建或编辑 Nginx 配置文件：

```bash
sudo nano /etc/nginx/sites-available/chaozhiyinqin.xyz
```

添加以下内容：

```nginx
server {
    listen 80;
    server_name chaozhiyinqin.xyz www.chaozhiyinqin.xyz;

    # 日志
    access_log /var/log/nginx/chaozhiyinqin.access.log;
    error_log /var/log/nginx/chaozhiyinqin.error.log;

    # 代理到 Flask
    location / {
        proxy_pass http://localhost:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # WebSocket 支持（如果需要）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        
        # 超时设置
        proxy_connect_timeout 600s;
        proxy_send_timeout 600s;
        proxy_read_timeout 600s;
    }

    # 静态文件（可选，如果直接提供静态文件）
    location /static {
        alias /var/www/html/static;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}
```

启用配置：

```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/chaozhiyinqin.xyz /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重新加载 Nginx
sudo systemctl reload nginx
```

## 🎯 最可能的原因

根据您的情况，最可能的原因是：

1. **Flask 应用没有运行** - 最常见
2. **Nginx 没有配置代理** - 需要配置反向代理
3. **防火墙阻止** - 需要开放端口
