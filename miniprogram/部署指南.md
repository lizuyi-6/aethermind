# AetherMind微信小程序部署指南

本指南将帮助您将AetherMind小程序部署到微信平台。

## 前置要求

1. **微信小程序账号**
   - 注册微信小程序账号：https://mp.weixin.qq.com/
   - 获取 AppID

2. **后端服务器**
   - 支持 HTTPS 的服务器
   - Flask 应用已部署并可访问
   - 已配置 CORS（允许微信小程序域名访问）

3. **开发工具**
   - 微信开发者工具：https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html

## 部署步骤

### 第一步：配置后端服务器

#### 1.1 确保服务器支持 HTTPS

微信小程序要求所有网络请求必须使用 HTTPS。您需要：

- 申请 SSL 证书（可以使用 Let's Encrypt 免费证书）
- 配置服务器支持 HTTPS
- 确保 Flask 应用可以通过 HTTPS 访问

#### 1.2 配置 CORS

修改 `app.py`，确保允许微信小程序访问：

```python
from flask_cors import CORS

app = Flask(__name__)
# 允许所有域名访问（生产环境建议限制为小程序域名）
CORS(app, resources={r"/api/*": {"origins": "*"}})
```

#### 1.3 测试后端 API

确保后端 API 可以正常访问：

```bash
curl https://your-server.com/api/config
```

### 第二步：配置小程序

#### 2.1 修改服务器地址

编辑 `miniprogram/app.js`：

```javascript
globalData: {
  baseUrl: 'https://your-server.com', // 修改为您的实际服务器地址
  // ...
}
```

#### 2.2 配置 AppID

编辑 `miniprogram/project.config.json`：

```json
{
  "appid": "wx1234567890abcdef", // 修改为您的 AppID
  // ...
}
```

### 第三步：配置微信公众平台

#### 3.1 登录微信公众平台

访问 https://mp.weixin.qq.com/ 并登录您的小程序账号。

#### 3.2 配置服务器域名

1. 进入"开发" -> "开发管理" -> "开发设置"
2. 找到"服务器域名"配置
3. 添加以下域名：

**request合法域名：**
```
https://your-server.com
```

**uploadFile合法域名：**
```
https://your-server.com
```

**downloadFile合法域名：**
```
https://your-server.com
```

**注意：**
- 域名必须使用 HTTPS
- 域名不能使用 IP 地址
- 域名不能使用端口号（除非是 443 端口）
- 需要先验证域名所有权

#### 3.3 配置业务域名（可选）

如果需要在小程序中打开网页，需要配置业务域名。

### 第四步：本地开发测试

#### 4.1 打开微信开发者工具

1. 打开微信开发者工具
2. 选择"导入项目"
3. 选择 `miniprogram` 目录
4. 输入 AppID（可以使用测试号）

#### 4.2 配置开发环境

在微信开发者工具中：

1. 点击右上角"详情"
2. 勾选"不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书"
   - 这样可以本地开发时访问本地服务器

#### 4.3 测试功能

- 测试聊天功能
- 测试文件上传
- 测试流式输出
- 检查错误日志

### 第五步：上传代码

#### 5.1 上传代码

1. 在微信开发者工具中点击"上传"
2. 填写版本号和项目备注
3. 点击"上传"

#### 5.2 提交审核

1. 登录微信公众平台
2. 进入"版本管理"
3. 找到刚上传的版本
4. 点击"提交审核"
5. 填写审核信息

**审核信息建议：**
- 功能描述：智能对话助手，支持文本对话和文件分析
- 服务类目：选择"工具" -> "效率工具"或"信息查询"
- 其他信息按要求填写

### 第六步：发布

审核通过后：

1. 在"版本管理"中找到审核通过的版本
2. 点击"发布"
3. 确认发布

## 常见问题

### Q1: 网络请求失败，提示"不在以下 request 合法域名列表中"

**解决方案：**
1. 检查是否在微信公众平台配置了服务器域名
2. 检查域名是否正确（必须包含 https://）
3. 开发环境可以勾选"不校验合法域名"进行测试

### Q2: 文件上传失败

**解决方案：**
1. 检查 uploadFile 合法域名是否配置
2. 检查文件大小是否超过 50MB
3. 检查文件类型是否支持

### Q3: 流式输出不流畅

**原因：** 小程序使用模拟流式输出（逐字符显示）

**解决方案：**
1. 可以调整输出速度（修改 `api.js` 中的间隔时间）
2. 如果后端支持 WebSocket，可以改用 WebSocket 实现真正的流式输出

### Q4: 本地开发无法连接服务器

**解决方案：**
1. 确保后端服务器正在运行
2. 检查服务器地址是否正确
3. 开发环境勾选"不校验合法域名"
4. 如果使用本地服务器，确保手机和电脑在同一网络

### Q5: HTTPS 证书问题

**解决方案：**
1. 使用有效的 SSL 证书
2. 确保证书未过期
3. 使用 Let's Encrypt 等免费证书服务

## 生产环境优化建议

1. **性能优化**
   - 使用 CDN 加速静态资源
   - 优化图片大小
   - 使用分包加载

2. **安全优化**
   - 限制 CORS 为小程序域名
   - 添加请求频率限制
   - 验证用户身份（如果需要）

3. **监控和日志**
   - 添加错误监控
   - 记录 API 调用日志
   - 监控服务器性能
