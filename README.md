# 智能体项目

一个本地智能体，能够对用户的自然语言做出回复，通过API调用大模型。

## 功能特性

- 🤖 支持多种大模型API（OpenAI、通义千问等）
- 💬 交互式命令行界面
- 📝 对话历史记录
- 🌊 流式输出模式
- ⚙️ 灵活的配置管理

## 安装

### Windows用户（推荐）

1. 克隆或下载项目到本地

2. 双击运行 `install_dependencies.bat` 安装依赖

3. 双击运行 `start_agent.bat` 启动智能体

### 手动安装

1. 克隆或下载项目到本地

2. 安装依赖：
```bash
pip install -r requirements.txt
```

## 配置

### 方式一：使用环境变量（推荐）

创建 `.env` 文件（参考 `.env.example`），或直接设置环境变量：

**使用OpenAI API：**
```bash
set MODEL_PROVIDER=openai
set OPENAI_API_KEY=your-api-key-here
set MODEL_NAME=gpt-3.5-turbo
```

**使用通义千问API：**
```bash
set MODEL_PROVIDER=tongyi
set DASHSCOPE_API_KEY=your-api-key-here
set MODEL_NAME=qwen-turbo
```

**使用自定义OpenAI兼容API：**
```bash
set MODEL_PROVIDER=custom
set CUSTOM_API_KEY=your-api-key-here
set API_BASE_URL=https://your-api-endpoint.com/v1
set MODEL_NAME=your-model-name
```

### 方式二：使用提示词文件

编辑 `system_prompt.txt` 文件来自定义系统提示词。程序会优先读取此文件。

### 方式三：修改config.py

直接编辑 `config.py` 文件中的默认值。

## 使用方法

### Web界面（推荐）✨

**Windows用户：**
- 双击 `start_web.bat` 文件启动Web服务
- 浏览器自动打开或手动访问 `http://localhost:5000`

**其他系统：**
```bash
python app.py
```
然后访问 `http://localhost:5000`

**Web界面功能：**
- 🎨 现代化的用户界面
- 💬 实时对话交互
- 📄 文件拖拽上传
- 🌊 流式输出支持
- 📝 对话历史管理
- 📱 响应式设计，支持移动端

### 命令行界面

**Windows用户：**

**方式一：一键启动（推荐）**
- 双击 `start_agent.bat` 文件即可启动

**方式二：命令行启动**
```bash
python agent.py
```

**其他系统：**

运行主程序：
```bash
python agent.py
```

### 交互命令

- 直接输入问题或对话内容，智能体会自动回复
- 输入 `quit` 或 `exit` 退出程序
- 输入 `clear` 清空对话历史
- 输入 `stream` 切换流式输出模式（逐字显示）
- 输入 `upload <文件路径>` 上传并分析文件
- 输入 `help` 查看帮助信息

### 文件上传功能

智能体支持上传和分析多种文件格式：

**支持的文件类型：**
- 文本文件：`.txt`, `.md`, `.markdown`
- PDF文档：`.pdf`
- Word文档：`.docx`, `.doc`
- Excel文件：`.xlsx`, `.xls`
- CSV文件：`.csv`
- JSON文件：`.json`
- 代码文件：`.py`, `.js`, `.java`, `.cpp`, `.c`, `.html`, `.css`

**使用方法：**
```
upload <文件路径>
upload <文件路径> <问题>
```

**示例：**
```

智能体: [分析PDF文件内容...]

你: upload data.xlsx 请帮我分析这个表格中的数据趋势

```

上传的文件会被复制到 `uploads/` 目录中，方便后续管理。

## 配置参数说明

| 环境变量 | 说明 | 默认值 |
|---------|------|--------|
| MODEL_PROVIDER | 模型提供商 (openai/tongyi/custom) | openai |
| OPENAI_API_KEY | OpenAI API密钥 | - |
| DASHSCOPE_API_KEY | 通义千问API密钥 | - |
| CUSTOM_API_KEY | 自定义API密钥 | - |
| API_BASE_URL | API基础URL | - |
| MODEL_NAME | 模型名称 | gpt-3.5-turbo |
| SYSTEM_PROMPT | 系统提示词（环境变量） | 默认使用"AetherMind"提示词 |
| TEMPERATURE | 温度参数 (0-2) | 0.7 |
| MAX_TOKENS | 最大token数 | 32000（已优化，支持生成约50万字的详细报告，通过多次续写完成） |

### SQLite-RAG（当前默认）

项目已迁移为默认使用本地 SQLite 向量库：

- 默认库路径：`knowledge_base/sqlite_rag.db`
- 可覆盖：`SQLITE_RAG_DB_PATH`
- 向量维度：`SQLITE_RAG_EMBED_DIM`（默认 `2048`）
- 候选池：`SQLITE_RAG_VECTOR_CANDIDATE_LIMIT`（默认 `300`）

示例（Windows）：
```bash
set SQLITE_RAG_DB_PATH=knowledge_base/sqlite_rag.db
set SQLITE_RAG_EMBED_DIM=2048
```

## 可行性研究报告模板

智能体在生成可行性研究报告时，**会自动加载并使用 `可行性研究报告模板.md` 文件**。

### 模板功能

- ✅ 自动检测：当用户要求撰写可行性研究报告时，系统会自动检测并加载模板
- ✅ 严格遵循：生成的报告严格按照模板的格式和结构
- ✅ 智能填充：将模板中的占位符（`{{}}`）替换为实际项目信息
- ✅ 完整结构：确保报告包含所有必需的章节和内容
- ✅ **详细生成**：系统会生成详细、完整、深入的报告内容（不少于500000字，约50万字）
- ✅ **数据完整**：所有表格、数据、分析都要详细填写，不能简略
- ✅ **自动续写**：当内容因上下文限制被截断时，系统会自动检测并续写，确保报告完整
- ✅ **自动保存**：生成的报告自动保存为MD文件到项目根目录
- ✅ **智能命名**：文件名包含项目名称和时间戳，避免覆盖

### 报告详细度保证

系统已优化，参考标准可行性研究报告格式，确保生成的报告内容详细、完整（详细报告标准，约50万字）：

- **字数要求**：报告总长度不少于500000字（约50万字），每个主要章节不少于70000字
- **行数要求**：报告总行数应达到12500-17500行，确保内容详实完整
- **数据要求**：所有数字都要具体，财务数据精确到万元并包含计算过程，市场数据引用具体来源（如XX协会、XX研究院）
- **表格要求**：所有表格完整填写，设备清单至少100-150项（含详细参数），收益预测至少10-20年（含详细分析），风险分析至少50-100种风险类型
- **章节完整**：确保所有章节都完整呈现，每个子章节都要详细展开（至少50-100个要点，每个要点1000-2000字），章节间有适当过渡
- **内容完整性**：每个章节都必须有详细内容，不能只有目录标题而没有具体内容，每个子章节必须包含至少3-5段详细内容
- **专业性**：包含详细的财务指标计算（IRR、NPV、投资回收期等）、SWOT分析、技术路线说明等，达到学术论文级别的详细程度

### 模板文件位置

模板文件：`可行性研究报告模板.md`

如果模板文件不存在，系统会提示用户，但仍可继续生成报告（建议使用模板以确保格式规范）。

### 报告保存功能

**自动保存规则：**
- 文件格式：Markdown (.md)
- 保存位置：项目根目录
- 文件命名：`{项目名称}_可行性研究报告_{时间戳}.md`
- 示例：`新能源项目_可行性研究报告_20241215_143022.md`

**文件路径输出：**
- 报告生成后，系统会自动显示保存的文件路径
- 支持普通模式和流式输出模式

### 自动续写功能

**智能续写机制：**
- ✅ **自动检测**：系统会自动检测生成的内容是否被截断
- ✅ **持续续写**：当检测到内容被截断时，自动调用续写功能，**持续续写直到报告完整**
- ✅ **多次续写**：支持最多20次续写，确保报告完整（即使受32K输出限制）
- ✅ **章节检查**：自动检查所有7个主要章节是否存在，确保不遗漏任何章节
- ✅ **内容连贯**：续写内容与前面部分保持连贯衔接
- ✅ **智能提示**：续写时会明确告知模型需要完成哪些缺失的章节

**续写触发条件：**
- API返回的finish_reason为"length"（达到token限制）
- 报告缺少任何必需的章节（检查所有7个主要章节）
- 最后章节（结论与建议）内容不完整（少于500字符或没有结束标记）
- 内容末尾没有完整的句子结束标记

**续写过程：**
1. 检测到截断后，自动分析缺失的章节
2. 提取最后4000字符作为上下文（增加上下文长度）
3. 发送续写请求，明确告知需要完成哪些章节
4. 将续写内容追加到原内容
5. 重复检测和续写，**直到所有章节完整或达到最大续写次数（20次）**
6. 显示续写进度和缺失章节信息

**续写保证：**
- 系统会持续续写，直到报告包含所有7个主要章节
- 即使受32K输出限制，也会通过多次续写完成整个报告
- 每次续写都会检查完整性，确保不遗漏任何部分

## 注意事项

1. 请妥善保管API密钥，不要将包含密钥的文件提交到公共仓库
2. 不同API提供商的计费方式不同，请注意使用成本
3. 对话历史会保存在内存中，程序退出后不会持久化
4. 上传的文件会被复制到 `uploads/` 目录，请注意文件大小和隐私
5. 处理大文件时可能需要较长时间，请耐心等待
6. 某些文件格式需要安装额外的依赖库（见requirements.txt）
7. Windows批处理脚本需要Python已添加到系统PATH环境变量中
